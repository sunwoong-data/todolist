## 발견된 버그

### BUG-01 | 공휴일 API 500 에러
- **심각도:** 중간 (Medium)
- **재현 조건:** 캘린더 뷰 전환 시 `/api/holidays` 호출
- **증상:** 공공데이터 API 프록시 500 Internal Server Error
- **영향:** 캘린더에 공휴일 표시 불가
- **원인 추정:** 공공데이터 API 외부 서버 장애 또는 `HOLIDAY_API_KEY` 유효성 만료
- **콘솔 오류:**
  ```
  GET http://localhost:3000/api/holidays?year=2026&month=5 → 500
  GET http://localhost:3000/api/holidays?year=2026&month=6 → 500
  ```
- **권장 조치:** 백엔드 `/api/holidays` 라우터에 외부 API 실패 시 빈 배열 반환하는 fallback 처리 추가

---

### BUG-02 | 로그인 실패 시 에러 메시지 미표시
- **심각도:** 중간 (Medium)
- **재현 조건:** 잘못된 이메일/비밀번호로 로그인 시도
- **증상:** 401 응답 수신 후 사용자에게 아무런 피드백 없음 (로그인 페이지 유지만 됨)
- **영향:** 사용자가 왜 로그인이 안 되는지 알 수 없음 (UX 불량)
- **권장 조치:** LoginPage에서 뮤테이션 `onError` 시 에러 메시지 state를 설정하고 폼 하단에 표시

---

### BUG-03 | 인증 만료/로그아웃 상태에서 CategoryFilter 크래시
- **심각도:** 높음 (High)
- **재현 조건:** 인증 토큰 만료 또는 로그아웃 상태에서 홈 화면 접근 시
- **증상:** `TypeError: categories.map is not a function` — CategoryFilter.tsx:159
- **영향:** 화면 전체 렌더링 실패 (화이트스크린 위험)
- **원인:** `useGetCategories` 훅이 401 에러 응답 시 배열이 아닌 값을 반환, CategoryFilter가 방어 처리 없이 `.map()` 호출
- **콘솔 오류:**
  ```
  TypeError: categories.map is not a function
      at CategoryFilter (CategoryFilter.tsx:159)
  ```
- **권장 조치:**
  ```typescript
  // CategoryFilter.tsx
  const safeCategories = Array.isArray(categories) ? categories : []
  ```
  또는 `useGetCategories` 훅에서 기본값을 `[]`로 보장

---

### BUG-04 | 삭제 확인 모달 인터셉션
- **심각도:** 낮음 (Low)
- **재현 조건:** 삭제 버튼 클릭 후 다른 요소 클릭 시도
- **증상:** 모달이 열린 상태에서 뒤 요소 클릭 차단 (의도된 동작이나 모달 닫기 안내 없음)
- **영향:** 모달 외부 클릭으로 닫을 수 없어 사용자 혼란 가능
- **권장 조치:** 모달 backdrop 클릭 시 닫기(취소) 동작 추가
