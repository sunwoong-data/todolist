## 1. 제품 개요

### 목적
인증된 사용자가 개인별로 할 일을 카테고리와 일정(시작일/종료일) 기반으로 관리할 수 있는 웹 애플리케이션

### 목표 사용자
- 학생: 과제, 시험 일정 등 학업 관련 할 일을 관리하는 사용자
- 직장인: 업무 태스크를 일정 기반으로 추적하고 싶은 사용자

### 비즈니스 목표
- 동시 접속 1,000명을 안정적으로 처리할 수 있는 서비스 제공
- 개인별 격리된 할 일 관리 환경 제공 (타인의 데이터 접근 불가)
- 시작일/종료일 기반 상태 자동 계산으로 별도 상태 관리 없이 직관적인 일정 파악

## 3. 기능 요구사항

### UC-01 회원가입 (BR-04)

**행위자**: 비인증 사용자

| 항목 | 내용 |
|------|------|
| 입력값 | 이메일(필수), 비밀번호(필수), 이름(필수) |
| 성공 시 | 계정 생성 + "기본" 카테고리 자동 생성(BR-04) |
| 실패 조건 | 중복 이메일 → 오류 반환, 계정 미생성 |

**수락 기준**
- 유효한 이메일/비밀번호/이름 입력 시 계정과 "기본" 카테고리가 함께 생성된다.
- 이미 가입된 이메일로 요청 시 오류가 반환되고 계정이 생성되지 않는다.

---

### UC-02 로그인 (BR-01)

**행위자**: 비인증 사용자

| 항목 | 내용 |
|------|------|
| 입력값 | 이메일(필수), 비밀번호(필수) |
| 성공 시 | JWT 인증 토큰 발급, 서비스 접근 허용 |
| v2 성공 시 추가 | 저장된 테마(theme_preference) 및 언어(language_preference) 로드하여 즉시 UI 적용 |
| 실패 조건 | 이메일 미존재 또는 비밀번호 불일치 → 인증 오류 반환 |

**수락 기준**
- 올바른 자격증명으로 로그인 시 토큰이 발급되고 서비스를 이용할 수 있다.
- 잘못된 비밀번호 입력 시 인증 오류가 반환된다.
- (v2) 로그인 성공 시 사용자가 이전에 설정한 테마(다크/라이트)가 즉시 적용된다.
- (v2) 로그인 성공 시 사용자가 이전에 설정한 언어(한국어/영어/일본어)가 즉시 적용된다.

|------|------|
| 수정 가능 항목 | 이름, 비밀번호 |

| 실패 조건 | 미인증 접근 → 인증 오류 반환 |

### UC-04 할 일 등록 (BR-03, BR-05, BR-06, BR-07)

**행위자**: 인증된 사용자

| 항목 | 내용 |
|------|------|
| 필수 입력 | 제목 |
| 선택 입력 | 설명, 카테고리, 시작일, 종료일 |
| 카테고리 미지정 시 | "기본" 카테고리 자동 배정 (BR-03) |
| 날짜 유효성 | 종료일 < 시작일이면 오류 반환 (BR-06) |
| 시작일/종료일 | 필수값 아님 (BR-07) |

**수락 기준**
- 카테고리 없이 등록 시 "기본" 카테고리로 자동 배정된다.
- 종료일이 시작일보다 이전이면 오류가 반환되고 할 일이 생성되지 않는다.

---

### UC-05 할 일 목록 조회 (BR-02, BR-08~BR-12)

**행위자**: 인증된 사용자

| 항목 | 내용 |
|------|------|
| 기본 조회 | 본인 소유의 할 일 전체 목록 |
| 필터 옵션 | 카테고리별(BR-08), 시작 전(BR-09), 진행 중(BR-10), 완료(BR-11), 기한 초과(BR-12) |
| 데이터 격리 | 타인의 할 일은 절대 포함되지 않음 (BR-02) |

**상태 계산 기준** (저장값 아닌 파생 계산값)

| 상태 | 조건 |
|------|------|
| 시작 전 | is_completed=false AND (start_date 없음 OR start_date > 오늘) |
| 진행 중 | is_completed=false AND start_date <= 오늘 AND (end_date 없음 OR end_date >= 오늘) |
| 기한 초과 | is_completed=false AND end_date < 오늘 |
| 완료 | is_completed=true |

**수락 기준**
- "진행 중" 필터 선택 시 조건에 맞는 항목만 반환된다.
- 타인의 인증 토큰으로 조회해도 해당 사용자의 할 일만 반환된다.

### UC-09 테마 설정 — 다크/라이트 모드 `[v2]`

**행위자**: 인증된 사용자

| 항목 | 내용 |
|------|------|
| 선택 옵션 | `light` (라이트 모드) / `dark` (다크 모드) |
| 저장 위치 | User 테이블 `theme_preference` 컬럼 (기본값: `light`) |
| 적용 시점 | 선택 즉시 UI 전체에 반영 |
| 유지 방법 | 로그인 시 서버에서 `theme_preference` 값을 반환, 클라이언트에서 즉시 적용 |

**User 모델 변경사항** (v2 추가 컬럼)

| 속성 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| theme_preference | String | `light` | 사용자 테마 설정 (`light` / `dark`) |

**수락 기준**
- Given 인증된 사용자가 다크 모드를 선택 시 / When 설정 저장 요청 시 / Then `theme_preference = dark`로 저장되고 UI가 즉시 다크 모드로 전환된다.
- Given 다크 모드를 저장한 사용자가 / When 다음 로그인 시 / Then 로그인 직후 다크 모드가 자동 적용된다.
- Given 라이트 모드로 변경 시 / When 설정 저장 요청 시 / Then `theme_preference = light`로 업데이트되고 UI가 라이트 모드로 전환된다.

---

### UC-10 언어 설정 — 다국어 지원 `[v2]`

**행위자**: 인증된 사용자

| 항목 | 내용 |
|------|------|
| 지원 언어 | `ko` 한국어 / `en` 영어 / `ja` 일본어 |
| 저장 위치 | User 테이블 `language_preference` 컬럼 (기본값: `ko`) |
| 적용 범위 | 앱 전체 UI 텍스트 (메뉴, 버튼, 레이블, 오류 메시지 등) |
| 적용 시점 | 선택 즉시 UI 전체 언어 전환 |
| 유지 방법 | 로그인 시 서버에서 `language_preference` 값을 반환, 클라이언트에서 즉시 적용 |

**User 모델 변경사항** (v2 추가 컬럼)

| 속성 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| language_preference | String | `ko` | 언어 설정 (`ko` / `en` / `ja`) |

**수락 기준**
- Given 인증된 사용자가 영어(`en`)를 선택 시 / When 저장 요청 시 / Then `language_preference = en`으로 저장되고 UI 전체가 즉시 영어로 전환된다 (BR-15)
- Given 일본어(`ja`)를 저장한 사용자가 / When 다음 로그인 시 / Then 로그인 직후 일본어 UI가 자동 적용된다 (BR-16)
- Given 유효하지 않은 언어 코드(`ko`/`en`/`ja` 외)로 / When 저장 요청 시 / Then 오류가 반환되고 기존 설정이 유지된다 (BR-15)
