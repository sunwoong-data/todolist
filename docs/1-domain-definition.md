## 1. 프로젝트 개요

**프로젝트명**: TodoList  
**목적**: 인증된 사용자가 개인별로 할 일을 카테고리와 일정(시작일/종료일) 기반으로 관리할 수 있는 웹 애플리케이션  
**핵심 문제**:
- 개인별로 격리된 할 일 관리 환경이 없음
- 할 일의 시작일과 종료일을 다루는 일정 관리 기능 부재

## 2. 핵심 도메인 개념 (용어 정의)

| 용어 | 정의 |
|------|------|
| 사용자 (User) | 회원가입 후 로그인한 인증된 개인. 자신의 할 일만 접근 및 관리 가능 |
| 할 일 (Todo) | 사용자가 등록하는 작업 단위. 제목, 설명, 시작일, 종료일, 상태, 카테고리를 가짐 |
| 카테고리 (Category) | 할 일을 분류하는 그룹. 사용자가 직접 생성하거나 기본 카테고리를 사용 |
| 기본 카테고리 | 할 일 등록 시 카테고리를 지정하지 않을 경우 자동으로 적용되는 카테고리 (이름: "기본") |
| 할 일 상태 (Status) | 할 일의 현재 진행 상태. 시작일/종료일 및 완료 여부에 따라 결정됨 |
| 시작일 (Start Date) | 할 일을 시작하는 날짜 |
| 종료일 (End Date) | 할 일을 완료해야 하는 목표 날짜 |
| 테마 (Theme) `[v2]` | 앱의 색상 모드. `light`(라이트 모드) 또는 `dark`(다크 모드) 중 하나. 사용자 계정에 저장되어 로그인 시 복원됨 |
| 언어 설정 (Language) `[v2]` | 앱 UI에 표시되는 언어. `ko`(한국어), `en`(영어), `ja`(일본어) 중 하나. 사용자 계정에 저장되어 로그인 시 복원됨 |

## 3. 도메인 모델 (주요 엔티티와 속성)

### User (사용자)

| 속성 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| email | String | 로그인 ID, 고유값 |
| password | String | 암호화된 비밀번호 |
| name | String | 사용자 이름 |
| theme_preference | String | 테마 설정 (`light` / `dark`, 기본값: `light`) `[v2]` |
| language_preference | String | 언어 설정 (`ko` / `en` / `ja`, 기본값: `ko`) `[v2]` |
| created_at | DateTime | 가입일시 |
| updated_at | DateTime | 정보 수정일시 |

### Category (카테고리)

| 속성 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| user_id | UUID | 카테고리 소유자 (User 참조) |
| name | String | 카테고리 이름 (동일 사용자 내 고유) |
| is_default | Boolean | 기본 카테고리 여부 |

### Todo (할 일)

| 속성 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| user_id | UUID | 할 일 소유자 (User 참조) |
| category_id | UUID | 분류 카테고리 (Category 참조) |
| assignee_id | UUID | 담당자 (Assignees 참조, 선택, ON DELETE SET NULL) `[신규]` |
| title | String | 할 일 제목 |
| description | String | 할 일 상세 설명 (선택) |
| start_date | Date | 시작일 (선택) |
| end_date | Date | 종료일 (선택) |
| is_completed | Boolean | 완료 여부 |
| created_at | DateTime | 등록일시 |
| updated_at | DateTime | 수정일시 |

### Todo 상태 (파생 값 - 저장하지 않고 계산)

| 상태 | 조건 |
|------|------|
| 시작 전 | is_completed = false AND (start_date가 없거나 start_date > 오늘) |
| 진행 중 | is_completed = false AND start_date <= 오늘 AND (end_date가 없거나 end_date >= 오늘) |
| 기한 초과 | is_completed = false AND end_date < 오늘 |
| 완료 | is_completed = true |

### Assignee (담당자) `[신규]`

| 속성 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| user_id | UUID | 담당자 소유자 (User 참조) |
| name | String | 담당자 이름 |
| avatar | Text | 프로필 이미지 URL (선택) |

### Anniversary (기념일) `[신규]`

| 속성 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| user_id | UUID | 기념일 소유자 (User 참조) |
| name | String | 기념일 이름 |
| month | SmallInt | 월 (1-12) |
| day | SmallInt | 일 (1-31) |

## 4. 비즈니스 규칙

### 인증 규칙
- **BR-01** 회원가입 후 로그인해야 앱 이용 가능
- **BR-02** 사용자는 자신의 데이터(할 일, 카테고리)에 대해서만 읽기/쓰기/삭제 가능

### 카테고리 규칙
- **BR-03** 카테고리를 지정하지 않고 할 일 등록 시, "기본" 카테고리로 자동 배정
- **BR-04** 각 사용자는 가입 시 "기본" 카테고리를 자동으로 보유

### 할 일 규칙
- **BR-05** 할 일의 등록, 수정, 삭제는 인증된 본인만 가능
- **BR-06** 종료일은 시작일보다 이전일 수 없음
- **BR-07** 시작일과 종료일은 필수값이 아님

### 필터링 규칙
- **BR-08** 카테고리별 필터링: 선택한 카테고리에 속한 할 일만 표시
- **BR-09** 시작 전 필터: 섹션 3의 "시작 전" 상태 조건에 해당하는 항목
- **BR-10** 진행 중 필터: 섹션 3의 "진행 중" 상태 조건에 해당하는 항목
- **BR-11** 완료 필터: 섹션 3의 "완료" 상태 조건에 해당하는 항목
- **BR-12** 기한 초과 필터: 섹션 3의 "기한 초과" 상태 조건에 해당하는 항목

### 테마 규칙 `[v2]`
- **BR-13** 테마 설정값은 `light` 또는 `dark` 만 허용하며, 기본값은 `light`
- **BR-14** 로그인 성공 시 서버는 `theme_preference` 값을 응답에 포함하고, 클라이언트는 이를 즉시 UI에 적용

### 언어 규칙 `[v2]`
- **BR-15** 언어 설정값은 `ko`, `en`, `ja` 만 허용하며, 기본값은 `ko`
- **BR-16** 로그인 성공 시 서버는 `language_preference` 값을 응답에 포함하고, 클라이언트는 이를 즉시 앱 전체 UI 언어에 적용

### 담당자 규칙 `[신규]`
- **BR-17** 각 사용자는 담당자 목록을 독립적으로 관리할 수 있으며, 할 일에 담당자를 할당 가능
- **BR-18** 할 일에서 담당자를 삭제하면 FK는 SET NULL로 처리 (담당자 자체는 유지)

### 기념일 규칙 `[신규]`
- **BR-19** 각 사용자는 기념일(생일, 기념일 등)을 월/일 기준으로 관리 가능
- **BR-20** 기념일은 할 일과 독립적인 자신만의 관리 도구

### 휴일 API 규칙 `[신규]`
- **BR-21** 공공 데이터 포털 휴일 API를 연동하여 연/월별 공휴일 정보 제공

### 수락 기준 (Acceptance Criteria)

#### UC-01 회원가입
- Given 비인증 사용자가 유효한 이메일, 비밀번호, 이름을 입력할 때 / When 회원가입 요청 시 / Then 계정이 생성되고 "기본" 카테고리가 자동 생성된다
- Given 이미 가입된 이메일로 / When 회원가입 요청 시 / Then 오류가 반환되고 계정이 생성되지 않는다

#### UC-02 로그인
- Given 등록된 이메일과 올바른 비밀번호로 / When 로그인 요청 시 / Then 인증 토큰이 발급되고 서비스 이용이 가능해진다
- Given 잘못된 비밀번호로 / When 로그인 요청 시 / Then 인증 오류가 반환된다

#### UC-04 할 일 등록
- Given 인증된 사용자가 카테고리 없이 할 일 등록 시 / When 등록 요청 시 / Then "기본" 카테고리로 자동 배정된다 (BR-03)
- Given 종료일이 시작일보다 이전 날짜로 / When 등록 요청 시 / Then 오류가 반환되고 할 일이 생성되지 않는다 (BR-06)

#### UC-05 할 일 목록 조회
- Given 인증된 사용자가 "진행 중" 필터 선택 시 / When 목록 조회 시 / Then start_date <= 오늘 AND end_date >= 오늘 AND is_completed = false 인 항목만 반환된다 (BR-10)
- Given 다른 사용자의 인증 토큰으로 / When 목록 조회 시 / Then 해당 사용자의 할 일만 반환되고 타인의 항목은 포함되지 않는다 (BR-02)

#### UC-07 할 일 삭제
- Given 인증된 사용자가 본인의 할 일을 / When 삭제 요청 시 / Then 해당 할 일이 영구 삭제된다
- Given 인증된 사용자가 타인의 할 일 ID로 / When 삭제 요청 시 / Then 권한 오류가 반환되고 삭제되지 않는다 (BR-05)

#### UC-09 테마 설정 `[v2]`
- Given 인증된 사용자가 다크 모드를 선택 시 / When 저장 요청 시 / Then `theme_preference = dark`로 저장되고 UI가 즉시 다크 모드로 전환된다 (BR-13)
- Given 다크 모드를 저장한 사용자가 / When 다음 로그인 시 / Then 로그인 직후 다크 모드가 자동 적용된다 (BR-14)
- Given 유효하지 않은 테마 값(`light` / `dark` 외)으로 / When 저장 요청 시 / Then 오류가 반환되고 기존 설정이 유지된다 (BR-13)

#### UC-10 언어 설정 `[v2]`
- Given 인증된 사용자가 영어(`en`)를 선택 시 / When 저장 요청 시 / Then `language_preference = en`으로 저장되고 앱 전체 UI가 즉시 영어로 전환된다 (BR-15)
- Given 일본어(`ja`)를 저장한 사용자가 / When 다음 로그인 시 / Then 로그인 직후 일본어 UI가 자동 적용된다 (BR-16)
- Given 유효하지 않은 언어 코드(`ko` / `en` / `ja` 외)로 / When 저장 요청 시 / Then 오류가 반환되고 기존 설정이 유지된다 (BR-15)
